Onboarding a New Client ~25 min per client

Use the onboarding generator tool to create the client in one click. No Supabase dashboard required — the API handles all table inserts.

Onboarding Tool

Open this file in your browser

docs/client-onboarding-generator.html
What the API creates

clients · client_profiles · hierarchy_nodes · daily_metrics · notifications · agent_activity

Link auth user (run in SQL Editor)
UPDATE client_profiles
SET user_id = '<auth_uuid_from_supabase>'
WHERE client_id = '<client_id_from_generator>';
Test lead — curl
curl -X POST https://app.auraflowusa.com/api/webhooks/lead \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <WEBHOOK_SECRET>" \
  -d '{
    "client_id": "<client_id>",
    "name": "Test Lead",
    "email": "test@example.com",
    "phone": "555-0100",
    "source": "angi",
    "service_requested": "Generator Installation",

06

mempalace Claude Code memory

mempalace gives Claude Code semantic memory of the codebase using ChromaDB + SQLite. After setup, Claude can search past decisions, architecture notes, and client context across sessions. Run these commands once in Terminal.

💡

These commands must be run manually in your terminal — they require interactive input and cannot be automated from Claude Code.

Step 1 — Install
pip3 install mempalace
Step 2 — Initialize (run in Terminal, not Claude Code)
python3 -m mempalace init /Users/motalebi/Downloads/auraflow-client-app
Step 3 — Index codebase
python3 -m mempalace mine
Step 4 — Register MCP server with Claude Code
claude mcp add mempalace -- python3 -m mempalace.mcp_server
✓

After setup: Claude Code has 19 memory tools — search, store, and retrieve knowledge about the codebase and clients across all sessions. The memory palace uses Wings → Rooms → Halls → Closets → Drawers → Tunnels hierarchy.

Local Development

Standard Next.js dev flow. Requires Node 18+ and npm.

First time setup
git clone <your-repo-url> auraflow-client-app
cd auraflow-client-app
npm install
cp .env.example .env.local
# Fill in all 13 env vars in .env.local
npm run dev
# Open http://localhost:3000
Day-to-day
npm run dev          # start dev server (hot reload)
npm run build        # production build — verify 0 errors before deploying
npm run lint         # check for type errors
Key directories
Project structure
src/app/(dashboard)/        — all client-facing pages
src/app/api/               — all API routes
src/components/ui/         — shared components
src/lib/intelligence/      — Claude bridge (server-only)
src/lib/supabase/          — Supabase clients (browser + service)
src/hooks/                 — React hooks
src/lib/store.ts           — Zustand global state
docs/                      — internal tools + schema SQL
worker/index.js            — service worker push handler
