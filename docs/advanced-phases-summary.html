One Table Still Needed

Run this in Supabase SQL Editor — all other tables already exist.

SQL — Supabase SQL Editor 
create table push_subscriptions (
  id         uuid primary key default gen_random_uuid(),
  client_id  text not null,
  user_id    uuid not null,
  endpoint   text not null unique,
  keys       jsonb not null,   -- { p256dh, auth }
  created_at timestamptz default now()
);

alter table push_subscriptions enable row level security;

create policy "Users manage their own push subscriptions"
  on push_subscriptions for all
  using (user_id = auth.uid());

create index on push_subscriptions (client_id);
Architecture

End-to-End Data Flow

── n8n workflow runs ──────────────────────────────────────────────────────

n8n Lead Scraper finds a new lead
  → POST /api/webhooks/lead  (Authorization: Bearer WEBHOOK_SECRET)
  → Upsert to lead_interactions
  → Log to agent_activity  (shows in live log instantly via Realtime)
  → Background: processNewLead() → Claude analysis → lead_analyses table
      → If urgency=urgent: POST /api/push/send → web push to client's phone
  → Client opens lead detail → LeadAnalysisPanel shows intent score + reply

n8n Metrics Sync (daily)
  → POST /api/webhooks/metric
  → Upsert daily_metrics → Home page metrics refresh on next load

n8n any workflow completes
  → POST /api/webhooks/agent
  → Insert agent_activity → Realtime → Agent live log updates

── client action from app ─────────────────────────────────────────────────

Client taps "Qualify →" on a lead card
  → PATCH /api/leads/[id] { status: "qualified" }
  → Optimistic UI updates instantly (no spinner)
  → qualified_at timestamp set + agent_activity logged

Client submits content request
  → POST /api/content { platform: "ig", brief: "..." }
  → triggerContentProduction() → notification + agent_activity + n8n webhook
  → Request appears in Content Engine history
